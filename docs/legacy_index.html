LatentMesh

Latent-space multi-agent communication for LLMs. Wire multiple agents together so they communicate entirely through KV caches and continuous hidden states — no text is ever passed between agents.

Installation

Install via uv:

bash 
pip install uv
uv pip install latentmesh

Quick Start

Build a three-agent pipeline (Plan → Reason → Review) in under 20 lines:

python 
from latentmesh import LatentLLM, PlanPrimitive, ReasonPrimitive, ReviewPrimitive
from latentmesh import LatentGraphState, decode_state
from langgraph.graph import StateGraph, START, END

llm = LatentLLM("Qwen/Qwen3-0.6B", device="cuda")

builder = StateGraph(LatentGraphState)
builder.add_node("planner",  PlanPrimitive(llm))
builder.add_node("reasoner", ReasonPrimitive(llm))
builder.add_node("reviewer", ReviewPrimitive(llm))

builder.add_edge(START, "planner")
builder.add_edge("planner", "reasoner")
builder.add_edge("reasoner", "reviewer")
builder.add_edge("reviewer", END)

graph = builder.compile()
result = graph.invoke({"messages": [{"role": "user", "content": "Prove that √2 is irrational."}]})
answer = decode_state(llm, result, [{"role": "assistant", "content": "Final answer: "}])

LatentLLM

Core model wrapper. Loads any HuggingFace causal LM and computes Wa at init.

LatentLLM(model_name, device="cuda", dtype="auto")
model_name
HuggingFace model identifier (e.g., "Qwen/Qwen3-0.6B")
device
Target device ("cuda", "cpu")
dtype
Torch dtype. "auto" uses native precision.

generate_latent()

generate_latent(messages=None, past_latent_state=None, num_thinking_steps=40) → LatentState

Runs num_thinking_steps of latent auto-regressive reasoning.

messages
Chat messages to encode as initial context. Each dict has "role" and "content".
past_latent_state
Continue from a prior agent's latent state. If provided, only trigger text is encoded.
num_thinking_steps
Number of recurrence steps in hidden-state space (default: 40).

Returns: LatentState with .kv_cache and .hidden_states.

LatentState

FieldTypeDescription
kv_cacheAnyTransformer KV cache (tuple of key/value per layer)
hidden_statesTensorHidden states from all recurrence steps [1, m, d_model]
debug_textstrHuman-readable label for logging

decode_state

Decodes text from a LatentState or graph state dict. The single point where continuous representations become tokens.

decode_state(llm, state, prompt, max_new_tokens=128, temperature=0.6, top_k=30000, top_p=0.95) → str
llm
The LatentLLM instance to use for decoding
state
A LatentState or a graph state dict with a "latent" key
prompt
Messages to prepend before generation (e.g., [{"role": "assistant", "content": "Final answer: "}])
temperature
Sampling temperature. Set to 0 for greedy decoding.
top_k
Top-K filtering threshold
top_p
Nucleus sampling threshold

Primitives

All primitives are callable LangGraph nodes (state → {"latent": LatentState}). Each uses a trigger text to steer the latent manifold before performing token-less recurrence.

AgentPrimitive (Base)

AgentPrimitive(name, llm, trigger_text="", num_thinking_steps=40)

Base class. Appends trigger text as an assistant message to shift the model's internal manifold, then runs generate_latent.

Built-in Primitives

PrimitiveTrigger TextPurpose
PlanPrimitive"Plan but don't execute: "Structural decomposition
ReasonPrimitive"Executing reasoning chain: "Core computation
ReviewPrimitive"Verify and check for errors: "Verification & error checking

VotingPrimitive

VotingPrimitive(name, candidates: List[AgentPrimitive])

Runs multiple candidates on deep-copied states. Selects winner by highest mean hidden-state L2 norm — higher norm = more settled reasoning = more confident trajectory.

Graph State

class LatentGraphState(TypedDict):
  messages: Annotated[List[dict], add]
  latent: Annotated[Optional[LatentState], latent_state_reducer]

Core state for LatentMesh LangGraph workflows. Compatible with StateGraph.

latent_state_reducer

Returns right if it exists, otherwise left. Since each agent's KV cache already contains all prior context, the latest state is always the most complete. Correct for sequential topologies.

merge_kv_caches(existing, incoming)

Fan-in utility. Concatenates KV caches along the sequence dimension (dim=2) for parallel worker results.

Example: Sequential Pipeline

Linear pipeline: Plan → Reason → Review. The most common topology for structured reasoning.

python · examples/sequential.py 
from latentmesh import (
    LatentLLM, PlanPrimitive, ReasonPrimitive,
    ReviewPrimitive, LatentGraphState, decode_state,
)
from langgraph.graph import StateGraph, START, END

llm = LatentLLM("Qwen/Qwen3-0.6B", device="cuda")

builder = StateGraph(LatentGraphState)
builder.add_node("planner",  PlanPrimitive(llm))
builder.add_node("reasoner", ReasonPrimitive(llm))
builder.add_node("reviewer", ReviewPrimitive(llm))
builder.add_edge(START, "planner")
builder.add_edge("planner",  "reasoner")
builder.add_edge("reasoner", "reviewer")
builder.add_edge("reviewer", END)

graph = builder.compile()
result = graph.invoke({
    "messages": [{"role": "user", "content": "Prove that √2 is irrational."}]
})

answer = decode_state(llm, result, [{"role": "assistant", "content": "Final answer: "}])
print(answer)

Example: Consensus Voting

Run multiple reasoning paths in parallel, select the best by latent certainty — no intermediate decoding.

python · examples/complex.py 
from latentmesh import (
    LatentLLM, ReasonPrimitive, VotingPrimitive,
    LatentGraphState, decode_state,
)
from langgraph.graph import StateGraph, START, END

llm = LatentLLM("HuggingFaceTB/SmolLM-135M", device="cpu")

candidates = [
    ReasonPrimitive(llm, trigger_text="Analytical approach: ", num_thinking_steps=24),
    ReasonPrimitive(llm, trigger_text="Creative approach: ",   num_thinking_steps=24),
    ReasonPrimitive(llm, trigger_text="Step-by-step: ",        num_thinking_steps=24),
]

consensus = VotingPrimitive("BestOfThree", candidates)

builder = StateGraph(LatentGraphState)
builder.add_node("vote", consensus)
builder.add_edge(START, "vote")
builder.add_edge("vote", END)

graph = builder.compile()
result = graph.invoke({"messages": [{"role": "user", "content": "Explain quantum entanglement."}]})
print(decode_state(llm, result, [{"role": "assistant", "content": "Answer: "}]))

Example: Hierarchical Routing

A supervisor agent makes routing decisions based on latent-state features — dynamic control flow without decoding.

python · examples/hierarchical.py 
# Supervisor examines latent-state norm to route to specialists
def route(state):
    latent = state["latent"]
    norm = latent.hidden_states.norm(dim=-1).mean().item()
    return "deep_thinker" if norm < threshold else "quick_solver"

builder.add_conditional_edges("supervisor", route, {
    "deep_thinker": "deep_thinker",
    "quick_solver": "quick_solver",
})

Example: FastAPI Deployment

Serve a latent graph as an HTTP endpoint. The model loads once at startup; each request runs the graph and decodes.

python · examples/deployment.py 
from contextlib import asynccontextmanager
from fastapi import FastAPI
import uvicorn

@asynccontextmanager
async def lifespan(app):
    app.state.llm = LatentLLM("HuggingFaceTB/SmolLM-135M", device="cpu")
    app.state.graph = build_graph(app.state.llm)
    yield

app = FastAPI(lifespan=lifespan)

@app.post("/generate")
async def generate(prompt: str):
    result = app.state.graph.invoke({
        "messages": [{"role": "user", "content": prompt}]
    })
    return {"response": decode_state(app.state.llm, result,
        [{"role": "assistant", "content": "Answer: "}])}
